⚡ Wan2GP Desktop

How to install & how it works — visual guide

1 Key insight

🔑
Wan2GP Desktop is a wrapper — it does NOT replace Wan2GP. All generation, models, scheduling, LoRAs run inside Wan2GP's own Gradio server. The desktop app adds: one-click install with GPU detection, environment management, embedded viewer with output file sidebar, metadata viewer, real-time terminal, and hardware tuning — everything you need to run Wan2GP without touching the command line.

2 Hardware-aware setup

🧠
Automatic GPU detection. The installer detects your exact GPU model (NVIDIA RTX 30/40/50, AMD, Apple Silicon) and selects the optimal wheels — correct PyTorch version, CUDA/ROCm, Flash Attention, Sage Attention, Triton, xFormers. RAM and VRAM are also detected and displayed on the dashboard. You just pick the environment type — everything else is automatic.

5 How it works

🖥️ Desktop App Electron — dashboard,
viewer, sidebar, terminal
→
🐍 Python Backend Wan2GP's Gradio server
on localhost:7860
→
🌐 Webview Embedded Chromium —
loads Wan2GP's UI
⚙️
Behind the scenes: The desktop app spawns wgp.py as a child process, waits for the Gradio server to start (TCP port check), then loads the URL into the embedded webview. All IPC goes through Electron's preload bridge. The output sidebar uses fs.watch for real-time file updates. Metadata is read by a brute-force binary JSON scanner (Node.js) with a Python fallback for Exif/ffprobe.

7 Sending files to Wan2GP

📤
Three ways to load generation settings into Wan2GP's "Load Settings from Media File" component:
🖱️
Drag-drop
Drag any file from the sidebar directly onto the Wan2GP UI
📤
Send / Vari / P+NS
Buttons below the file list → creates inline JSON from cached metadata → drops on component
📋
Click metadata row
Click any row in the metadata panel → sends the file to Wan2GP
All three trigger the same pipeline: cached metadata → JSON file → synthetic drop on Gradio's file component → Wan2GP reads it → updates model, prompt, seed, steps, guidance, resolution — everything.
